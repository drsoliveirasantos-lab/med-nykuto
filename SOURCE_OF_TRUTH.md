# Med Nykuto — source of truth

This file is the first reference to read before editing the repository.

## Course and module content

Authoritative editable sources:

```txt
content/courses/
```

Each course declares its module count and module order in:

```txt
content/courses/<course>/course.json
```

The locked expected counts live in:

```txt
content-lock.json
```

## Generated course runtime data

Generated file used by the browser:

```txt
data/med-courses-data.js
```

Do not edit this file manually. Edit `content/courses/**`, then rebuild:

```bash
node scripts/build-courses-data.js
```

## App bundle source

The runtime bundle remains:

```txt
app.bundle.js
```

Editable split sources are under:

```txt
src/app-bundle/
src/i18n/app-bundle/
src/dom/app-bundle/
```

Do not edit `app.bundle.js` directly for application logic. Edit split sources, rebuild, then validate.

## Validation commands

Before merging content/source changes, run:

```bash
node scripts/validate-course-sources.js
node scripts/validate-no-stale-files.js
npm run validate
```

GitHub Actions must be green before merge.

## Files that are not authoritative

Do not use old migration files, raw extraction files or backups as current truth.

Forbidden stale names include:

```txt
README_SPLIT_MIGRATION.md
split_migration_manifest.json
_source_backup/med-courses-data-live-v363-original-58.js
window.txt
```

If a historical file must be kept, place it under `docs/archive/` and mark it clearly as non-runtime history.

## Safe editing rule

1. Edit source files only.
2. Rebuild generated files when needed.
3. Run validation.
4. Open a small PR.
5. Merge only after checks are green.
